# Excel AI 编辑器中文使用说明

这是一份面向实际使用的完整说明。你可以把它理解成插件的“使用手册”：从安装、插入表格、导入导出，到 AI 模型配置、选区处理、公式计算和常见问题，都放在这里。

## 一句话介绍

Excel AI 编辑器可以在思源笔记当前页面中插入一个可编辑的 Excel 表格，并让 AI 对选中的表格区域逐行处理。AI 结果会写入新增列，不会覆盖原始数据。

## 适合场景

- 阅读摘录：把书籍、文章、课程笔记整理成表格，再用 AI 总结重点。
- 资料清单：批量提取姓名、电话、标签、书名、作者、地点等信息。
- 选题管理：把标题、摘要、来源放入表格，用 AI 分类或改写。
- 课程与内容运营：对学员反馈、主题库、文案库做归类、提取和补全。
- 本地知识处理：配合 Ollama 或 LM Studio，在本机完成表格 AI 处理。

## 安装方式

### 从思源集市安装

1. 打开思源笔记。
2. 进入 `集市`。
3. 搜索 `Excel AI 编辑器`。
4. 点击下载并启用插件。
5. 在思源顶部栏找到表格按钮。

### 手动安装

1. 从 GitHub Release 下载 `package.zip`。
2. 解压后确认目录名为 `siyuan-excel-ai`。
3. 放入思源插件目录：

```text
SiYuan/data/plugins/siyuan-excel-ai
```

4. 重启思源。
5. 在 `集市 - 已下载` 中启用插件。

## 快速开始

1. 打开任意一篇思源笔记。
2. 点击顶部栏的 Excel AI 图标。
3. 当前笔记中会插入一个 Excel 表格块。
4. 你可以直接编辑表格，也可以点击 `导入 .xlsx`。
5. 用鼠标选择需要 AI 处理的区域。
6. 点击 `AI 应用` 打开右侧面板。
7. 选择 AI 应用类型。
8. 点击 `预览前 3 行` 检查效果。
9. 确认无误后点击 `应用到选区`。
10. 修改完成后点击顶部栏 `保存`，当前表格块的数据会覆盖保存。

## 表格编辑

插件内置 Luckysheet 表格编辑器，支持常见的表格操作：

- 输入、复制、粘贴单元格内容。
- 调整字体、字号、加粗、斜体、下划线。
- 设置单元格颜色、对齐、边框。
- 切换工作表。
- 选择行、列或连续区域。
- 导入 `.xlsx`。
- 导出 `.xlsx`。
- 保存当前工作簿数据。

当前版本优先保证 `.xlsx` 的内容读写。复杂样式、宏、图表、透视表和高级对象不作为第一版兼容重点。

## 保存与恢复

每个插入到笔记里的 Excel 表格块都会有独立存档。

- 点击顶部栏 `保存` 后，插件会通过思源内核文件 API 保存当前工作簿的最新数据。
- 关闭或重启思源，再次打开同一篇笔记时，表格会自动恢复到上次保存的状态。
- 同一篇笔记里插入多个 Excel 表格块时，它们会分别保存，互不覆盖。
- 保存会覆盖这个表格块的上一份存档。
- 如果只是编辑后关闭页面但没有点击 `保存`，未保存的修改不会写入存档。

建议在这些操作后点击一次 `保存`：

- 手动改完单元格内容。
- 导入新的 `.xlsx` 文件。
- 使用 AI 写入结果列。
- 调整表格结构或工作表内容。

## AI 应用类型

### 内容总结

适合处理长文本、摘录、评论、反馈、说明文字。

示例：

```text
把 B 列每一行的长文本压缩成一句摘要
```

默认结果列名：`AI 总结`

### 自定义 AI

适合按你的指令处理每一行。你可以让 AI 改写、润色、翻译、补充、压缩、提取要点。

示例：

```text
把这一行内容整理成适合朋友圈发布的一句话，语气温柔、克制、清晰
```

默认结果列名：`AI 结果`

### 智能分类

适合把每行数据归到指定分类中。

候选分类示例：

```text
工作, 生活, 学习, 灵感, 待处理
```

默认结果列名：`AI 分类`

### 智能提取

适合从文本中提取结构化信息。

示例：

```text
提取姓名、电话、标签
提取书名和作者
提取地点、时间和事件
```

默认结果列名：`AI 提取`

### 公式计算

适合做简单数字计算。公式会在本地执行，AI 只用于“自然语言补全公式”。

公式示例：

```text
={单价}*{数量}
ROUND(AVG({语文},{数学}), 1)
SUM({收入},-{支出})
```

支持函数：

```text
SUM, AVG, MIN, MAX, ROUND, ABS, COUNT, POW, SQRT
```

默认结果列名：`公式结果`

## 选区规则

AI 会按“行”处理当前选中的表格区域。

- 勾选 `选区第一行是表头` 时，第一行会作为字段名，不参与 AI 批处理。
- 未勾选时，插件会使用列字母作为字段名，例如 `A`、`B`、`C`。
- 每次运行都会新增一个结果列。
- 如果结果列名已经存在，会自动追加序号，避免覆盖旧结果。

建议把表格整理成这种结构：

```text
第一行：字段名
后续行：每条数据一行
```

## 模型配置

点击 `AI 模型设置` 可以填写模型服务。

### 本地 Ollama

推荐配置：

```json
{
  "preset": "ollama",
  "protocol": "ollama",
  "endpoint": "http://127.0.0.1:11434",
  "model": "qwen3:8b",
  "apiKey": ""
}
```

如果模型为空，可以点击 `检测可用模型`。

### LM Studio

常见配置：

```json
{
  "preset": "lmstudio",
  "protocol": "openai",
  "endpoint": "http://127.0.0.1:1234/v1",
  "model": "local-model",
  "apiKey": ""
}
```

### OpenAI-compatible 接口

适用于 OpenAI、DeepSeek、Kimi、智谱 GLM、阿里云百炼 / 通义千问、硅基流动、OpenRouter 或自建接口。

常见配置：

```json
{
  "preset": "custom",
  "protocol": "openai",
  "endpoint": "https://你的接口地址/v1",
  "model": "你的模型名",
  "apiKey": "你的 API Key"
}
```

### Anthropic Claude

选择 Anthropic 预设后填写 Claude API 地址、模型名和 API Key。

### Google Gemini

选择 Gemini 预设后填写 Gemini API 地址、模型名和 API Key。

## 网络与隐私

- `localhost`、`127.0.0.1`、`::1` 会直接请求。
- 远程 API 会通过思源 `/api/network/forwardProxy` 转发，避免浏览器跨域问题。
- 使用第三方 API 时，选中的表格内容会发送给对应服务商。
- API Key 只保存在当前思源客户端的插件数据中。
- 插件不会主动上传整篇笔记，只处理你选中的表格区域。

## 推荐工作流

1. 先导入或整理表格。
2. 确认第一行是否为表头。
3. 选择要处理的数据区域。
4. 选择 AI 应用类型。
5. 用 `预览前 3 行` 检查提示词和模型输出。
6. 不满意就调整指令、分类或模型。
7. 满意后再 `应用到选区`。
8. 导出 `.xlsx` 或继续在思源中保留表格。

## 常见问题

### 点击 AI 后没有结果

请检查：

- 是否已经选择表格区域。
- 是否已经配置模型。
- 模型服务是否正在运行。
- 本地模型地址是否正确。
- 远程 API Key 是否有效。

### Ollama 检测不到模型

请确认 Ollama 已启动，并在终端执行过：

```text
ollama list
```

如果服务地址不是默认地址，请在模型设置中改成实际地址。

### AI 结果为空

可能原因：

- 模型返回内容为空。
- 指令过于模糊。
- 选区中没有有效文本。
- API 调用失败或被限流。

建议先用 `预览前 3 行` 测试。

### 表格导出后样式不完全一致

当前版本优先保证内容导入导出。复杂样式、宏、图表等高级功能暂不保证完全还原。

### 会不会覆盖原数据

不会。AI 结果默认写入新增结果列。如果同名列已存在，会自动生成带序号的新列名。
